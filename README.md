# Application réservation appartement — version sécurisée Vercel + Supabase

Correction principale : les réservations ne sont plus lues directement depuis le navigateur public.

## Fichiers

- `index.html` : page publique visiteur.
- `admin.html` : page propriétaire.
- `api/*` : fonctions serveur Vercel.

## Variables Vercel à créer

Dans Vercel > Project Settings > Environment Variables :

- `SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY`
- `ADMIN_PASSWORD`
- `SESSION_SECRET`

Ne jamais mettre la clé `service_role` dans un fichier HTML.

## SQL Supabase

```sql
create table if not exists reservations (
  id bigint generated by default as identity primary key,
  date text,
  date_label text,
  time text,
  prenom text,
  nom text,
  tel text,
  email text,
  situation text,
  groupe text,
  message text,
  created_at timestamptz default now()
);

create table if not exists weekend_requests (
  id bigint generated by default as identity primary key,
  nom text,
  tel text,
  email text,
  jour text,
  heure_debut text,
  heure_fin text,
  message text,
  created_at timestamptz default now()
);

create table if not exists app_config (
  key text primary key,
  value jsonb
);
```

## Déploiement

1. Créer un dépôt GitHub.
2. Mettre tous les fichiers du dossier dans GitHub.
3. Importer dans Vercel.
4. Ajouter les variables d'environnement.
5. Déployer.

Page publique : `/`  
Page propriétaire : `/admin.html`
